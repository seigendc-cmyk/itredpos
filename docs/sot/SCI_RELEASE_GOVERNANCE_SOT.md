# SCI Release Governance SOT

**Owner:** Digital Commerce (Private) Limited  
**Status:** Controlled Source of Truth  

## Purpose

This SOT defines who may approve, publish, deploy and withdraw an SCI release.

## Release flow

Business Requirement
-> Approved SOT
-> Architecture
-> Implementation
-> Technical Review
-> Testing
-> Security Review
-> Release Approval
-> Git Tag
-> GitHub Release
-> Deployment
-> Post-Deployment Validation

## Approval decisions

- APPROVED FOR RELEASE
- APPROVED WITH DOCUMENTED LIMITATIONS
- CORRECTION REQUIRED
- RELEASE BLOCKED

## Authority rule

No developer, contractor, coding agent or support operator may independently
approve or publish a production SCI release.
