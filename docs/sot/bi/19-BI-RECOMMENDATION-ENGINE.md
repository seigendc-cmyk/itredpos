# SCI POS Business Intelligence Recommendation Engine

## Purpose

The Recommendation Engine turns BI findings into practical business action.

Alerts tell the user that something needs attention.

Recommendations tell the user what to do next.

SCI POS must not only detect risk. It must guide vendors toward better decisions.

## Core Principle

Every recommendation must be simple, explainable, and actionable.

A recommendation must answer:

- What did the system observe?
- Why does it matter?
- What is the risk?
- What action should be taken?
- What result is expected?

## Recommendation Format

SCI POS must use a consistent structure:

Observation:
What the system noticed.

Reason:
Why the system believes it happened.

Risk:
What could go wrong if ignored.

Recommended Action:
What the user should do.

Expected Benefit:
What improves if action is taken.

## Recommendation Types

### Stock Recommendations

Examples:

- Count this shelf today
- Investigate this product variance
- Transfer stock from one branch to another
- Stop selling into negative stock
- Increase count frequency for high-risk products

### COGS Recommendations

Examples:

- Increase selling price
- Review supplier cost
- Stop selling below cost
- Update landed cost
- Protect target margin

### Profit Recommendations

Examples:

- Review discount activity
- Reduce write-offs
- Investigate protected profit decline
- Review weak-margin categories

### Cash Recommendations

Examples:

- Review cashier cash variance
- Confirm delivery cash remittance
- Restrict drawer opening permissions
- Investigate shift shortage

### Staff Recommendations

Examples:

- Review staff transaction activity
- Provide training
- Reduce override permission
- Assign supervisor review

### Supplier Recommendations

Examples:

- Compare alternative suppliers
- Review invoice mismatch
- Increase receiving checks
- Renegotiate pricing

### Delivery Recommendations

Examples:

- Review driver remittance
- Investigate route deviation
- Verify failed delivery
- Improve delivery assignment rules

## Recommendation Priority

Recommendations must be prioritized by:

- Financial impact
- Risk severity
- Urgency
- Repetition
- Business area affected
- Owner role relevance

Critical recommendations must appear first.

## Future Prediction Recommendations

SCI POS must eventually recommend action before the problem happens.

Examples:

- Reorder before stockout
- Adjust price before margin collapse
- Contact customer before churn
- Count shelf before expected variance
- Review supplier before cost increase damages margin

## Strategic Principle

The Recommendation Engine is where SCI POS begins to act like a business advisor.

The system must help ordinary vendors move from seeing information to taking action.
